# ClassForge ローカル動作確認手順

> 作成日: 2026-05-07  
> 対象バージョン: Phase 1 全完了 + Phase 2（優先度：高）全完了  
> 所要時間目安: 約 30〜40 分

---

## 事前準備チェックリスト

- [x] Node.js 22 LTS がインストールされている
- [x] `.env` に `DATABASE_URL` と `DIRECT_URL` が設定されている
- [x] `.env` に `AUTH_SECRET` が設定されている
- [ ] （決済テスト時）`.env` に `STRIPE_SECRET_KEY`、`STRIPE_WEBHOOK_SECRET` が設定されている
- [x] （サムネイルテスト時）`.env` に `SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY` が設定されている
- [x] Stripe CLI がインストールされている（`stripe --version` で確認）

---

## STEP 1 — ローカル環境の起動準備

### 1-1. 依存パッケージのインストール

```bash
cd class-forge
npm install
```

- [x] エラーなく完了する

### 1-2. Prisma クライアントの生成

```bash
npx prisma generate
```

- [x] `Generated Prisma Client` が表示される

### 1-3. シードデータの投入（DBが空のとき）

> **注意:** シードを実行すると既存データは削除・上書きされます。初回のみ実行してください。

```bash
npm run seed
```

投入されるデータ:

| 種別 | 内容 |
|------|------|
| 管理者アカウント | `admin@classforge.jp` / `admin123` |
| 講座① | ChatGPT仕事術入門（¥9,800）- 4セクション / 16レッスン |
| 講座② | 副業AI活用講座（¥14,800）- 5セクション / 19レッスン |
| 講座③ | 非エンジニア向け自動化入門（無料）- 3セクション / 11レッスン |

- [x] `Seeding completed` または `Done` が表示される

### 1-4. 開発サーバーの起動

```bash
npm run dev
```

- [ ] `Local: http://localhost:3000` が表示される
- [ ] ブラウザで `http://localhost:3000` を開き、トップページが表示される

### 1-5. Stripe Webhook のローカル転送（決済テスト時のみ）

> 別のターミナルで実行してください。

```bash
stripe listen --forward-to http://localhost:3000/api/webhook
```

- [ ] `Ready! Your webhook signing secret is whsec_...` が表示される
- [ ] 表示された `whsec_...` の値を `.env` の `STRIPE_WEBHOOK_SECRET` にセットして、`npm run dev` を再起動する

## STEP 2 — 管理者フローの確認

### 2-1. 管理者ログイン

1. `http://localhost:3000/login` にアクセス
2. 以下の認証情報でログイン

   | 項目 | 値 |
   |------|-----|
   | メールアドレス | `admin@classforge.jp` |
   | パスワード | `admin123` |

- [ ] ログイン後、トップページ（`/`）にリダイレクトされる
- [ ] ヘッダーに「ダッシュボード」リンクが表示される

### 2-2. 管理ダッシュボードの確認

`http://localhost:3000/admin` にアクセス

- [ ] 「会員数」「講座数」「購入数」「総売上」の集計カードが表示される
- [ ] シードデータの 3 講座分が反映されている（講座数 ≧ 3）

### 2-3. 新規講座の作成

`http://localhost:3000/admin/courses/new` にアクセス

1. 以下を入力して「講座を作成」をクリック

   | フィールド | 入力値（例） |
   |-----------|------------|
   | 講座タイトル | `テスト講座 2026` |
   | スラッグ | 自動生成されることを確認（手動変更も可） |
   | 説明文 | 任意のテキスト |
   | 価格 | `0`（無料）または `1000` |
   | ステータス | 下書き |

- [ ] 送信後、`/admin/courses/[courseId]` の編集画面にリダイレクトされる

### 2-4. サムネイル画像アップロードのテスト ★ Phase 2 新機能

> **前提:** `.env` の `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` が設定済みで、Supabase Dashboard に `course-thumbnails`（Public）バケットが作成されていること。

`/admin/courses/[courseId]` の基本情報フォーム上部にある「サムネイル画像」エリアで確認する。

1. 点線枠のエリアをクリックし、JPEG・PNG・WebP・GIF のいずれかの画像を選択する

- [ ] 選択直後にローカルプレビューが即時表示される（楽観的更新）
- [ ] 「アップロード中...」のスピナーオーバーレイが表示される
- [ ] 完了後、「サムネイルを更新しました」の成功メッセージが表示される
- [ ] 「画像を変更する」リンクが表示され、再クリックで別の画像に差し替えられる

> **Supabase 未設定の場合:** 「Supabase 環境変数が設定されていません」エラーが表示されることを確認する（クラッシュしないこと）

### 2-5. セクション・レッスンの追加

`/admin/courses/[courseId]` のカリキュラムセクションで確認する。

#### セクション追加

1. 画面下部の「新しいセクション名...」に `第1章：はじめに` と入力し「セクションを追加」をクリック
2. 続けて `第2章：基礎知識` を追加する

- [ ] セクションがリストに追加される
- [ ] 各セクションの左端にドラッグハンドル（⠿ アイコン）が表示される

#### セクション並び替え（D&D）★ Phase 2 新機能

1. `第2章：基礎知識` のドラッグハンドルを掴み、`第1章：はじめに` の上にドロップする

- [ ] ドラッグ中にカードが半透明になり、位置が入れ替わる
- [ ] ドロップ後、即座に表示順が変わる（楽観的更新）
- [ ] ページをリロードしても並び順が保持されている（DB に保存されている）
- [ ] ネットワークを無効にした状態でD&Dを実行すると、元の順序にロールバックされ赤いエラーメッセージが表示される

#### レッスン追加

1. セクションの「＋ レッスンを追加」をクリック
2. `レッスン1：概要` と入力して「追加」をクリック

- [ ] レッスンが追加される
- [ ] レッスン編集リンク（`/admin/courses/[courseId]/lessons/[lessonId]`）が表示される

### 2-6. 講座の公開

`/admin/courses/[courseId]` の基本情報フォームで確認する。

1. ステータスを「公開」に変更し「保存する」をクリック

- [ ] 「保存しました」のメッセージが表示される
- [ ] `http://localhost:3000/courses` に作成した講座が表示される（ステータスが PUBLISHED のもののみ一覧表示）

視聴ページで 30 秒以上待機する（または DevTools → Network タブを確認）

- [ ] 30秒ごとに `POST /api/progress` リクエストが発生する（Network タブで確認）
- [ ] ページを再読み込みしても同じレッスンの途中から再開できる（`initialSeconds` が設定されている）

### 3-8. 手動完了マークボタンのテスト ★ Phase 2 新機能

視聴ページのレッスンタイトル右側に表示されるボタンで確認する。

#### 完了 → 未完了

2. 「完了済み」ボタンを再クリック

- [ ] ボタンが「完了にする」（アウトライン style）に戻る
- [ ] サイドバーの ✓ バッジが数字アイコンに戻る

### 3-9. マイページの確認

`http://localhost:3000/mypage` にアクセス

- [ ] URL からクエリパラメータがすべて消える

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| `prisma generate` がエラー | `DATABASE_URL` の接続文字列を確認 |
| シードが失敗する | DBへの接続確認・既存データの競合がないか確認 |
| ログインできない | シードが完了しているか・パスワードが `admin123` であるか確認 |
| Stripe 購入後に `/thanks` に遷移しない | Stripe CLI の Webhook 転送が起動中か確認 |
| サムネイルアップロードが失敗する | `.env` の `SUPABASE_SERVICE_ROLE_KEY` と Supabase バケット（`course-thumbnails`）の設定を確認 |
| D&D が反応しない | ドラッグハンドル（左端の ⠿ アイコン）を掴んでいるか確認。編集中のセクションは掴めないので「キャンセル」後に試す |
| 完了マークボタンが表示されない | 視聴ページの URL が `/learn/[courseSlug]/[lessonSlug]` の形式になっているか確認 |
