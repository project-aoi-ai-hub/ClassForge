## 静的解析で発見した問題点

### BUG-01 【重大】手順書 3-10: パスワード例がバリデーション要件を満たさない

- **種別:** 手順書と実装の不整合（実装バグ or ドキュメントバグ）
- **場所:** `src/app/(authenticated)/account/actions.ts:74`
- **内容:** `updatePassword` アクションは「英大文字・数字・記号を各1文字以上含む8文字以上」を要求するが、手順書 3-10 の新パスワード例 `newpassword123` は英大文字・記号を含まないためバリデーションエラーになる
- **エラーメッセージ:** `パスワードには英大文字、数字、記号をそれぞれ1文字以上含める必要があります`
- **再現方法:** STEP 3-10 にて `newpassword123` を新パスワードとして入力して送信
- **影響:** テスト手順書の通りに実施するとパスワード変更テストが失敗する

### BUG-02 【重大】手順書 3-5: 無料講座受講フローの記述が実装と一致しない

- **種別:** 手順書と実装の不整合（動作フロー誤記）
- **場所:** `src/app/(public)/courses/[slug]/page.tsx:171`, `src/app/(authenticated)/checkout/[courseId]/page.tsx`
- **内容:** 手順書には「無料で受講するボタンをクリックすると `/learn/no-code-automation` にリダイレクトされる（決済不要）」と記述されているが、実際のフローは異なる:
  1. 「無料で受講する」ボタン → `/checkout/[courseId]`（購入確認ページ）
  2. 購入確認ページで「無料で受講を開始する」をクリック
  3. API が購入記録を作成し `/thanks` ページへリダイレクト
- **期待動作:** ワンクリックで `/learn/[courseSlug]` へ遷移するか、`/thanks` で「マイページへ」から学習開始
- **影響:** テスト手順書通りに確認するとチェック項目に失敗する

### BUG-03 【中程度】手順書 3-3: Course 2 のスラッグが間違っている

- **種別:** ドキュメントの誤記
- **場所:** 手順書 STEP 3-3 vs `prisma/seed.ts:119`
- **内容:** 手順書には `ai-side-business` と記載されているが、シードデータのスラッグは `side-business-ai`
- **再現方法:** `http://localhost:3000/courses/ai-side-business` にアクセスすると 404
- **影響:** 手順書のスラッグを直接コピペしてアクセスすると 404 になる（注釈に「実際のスラッグは講座一覧のURLで確認してください」とあるため軽微）

### BUG-04 【中程度】VideoPlayer: 一時停止中も視聴秒数がカウントアップされる

- **種別:** ロジックバグ（精度問題）
- **場所:** `src/components/player/VideoPlayer.tsx:57-68`
- **内容:** 進捗の自動保存は `setInterval` で30秒ごとに `watchedRef.current += 30` を実行しているが、これは「ページを開いてからの経過時間」であり「実際に動画を視聴した秒数」ではない。動画を一時停止していても、ページを開いているだけでDBの視聴秒数が増加し続ける
- **コード抜粋:**
  ```ts
  progressTimer.current = setInterval(() => {
    watchedRef.current += 30;  // ← 実際の再生状態に関係なく加算
    saveProgress(watchedRef.current);
  }, 30000);
  ```
- **影響:** 「30秒後のresumeポイント」の精度が低い。完了率やマイページの進捗バーが実際の視聴状況と乖離する可能性

### BUG-05 【軽微】売上管理: 日付フィルターのタイムゾーン問題

- **種別:** 実装バグ（本番環境で顕在化）
- **場所:** `src/app/admin/orders/page.tsx:32-36`
- **内容:** `new Date(\`${from}T00:00:00\`)` はサーバーのシステムタイムゾーンで解釈される。本番サーバーがUTCで動作している場合、JST（UTC+9）との9時間ずれが生じ、日付フィルターが正しく機能しない
- **影響:** ローカル開発環境では問題が顕在化しにくいが、本番環境（UTC）では期待通りにフィルタリングされない可能性

### BUG-06 【軽微】checkout API: `NEXT_PUBLIC_APP_URL` 未設定時に無効なURLが生成される

- **種別:** フォールバック欠如
- **場所:** `src/app/api/checkout/route.ts:52`
- **内容:** `${process.env.NEXT_PUBLIC_APP_URL}/thanks?free=true` で環境変数が未設定の場合、`undefined/thanks?free=true` という無効なURLが生成される
- **影響:** `.env` に設定されていれば問題ないが、設定漏れ時にサイレントに不正なリダイレクトが発生する

### INFO-01 【情報】thanks ページが `?free=true` パラメータを使用していない

- **場所:** `src/app/(public)/thanks/page.tsx`
- **内容:** `checkout.ts` は無料受講時に `/thanks?free=true` を返すが、`thanks/page.tsx` はこのパラメータを一切使用していない。有料購入と無料受講で同一内容のページが表示される
- **影響:** 機能上の問題はないが、UX上は無料受講完了と有料購入完了のメッセージが区別されていない

### INFO-02 【情報】mypage.tsx の認証チェックが他ページと不一致

- **場所:** `src/app/(authenticated)/mypage/page.tsx:18`
- **内容:** `if (!session?.user?.id) return null;` → 他のページは `redirect("/login")` を使用。ただし親レイアウトで既にリダイレクトされるため実害なし
